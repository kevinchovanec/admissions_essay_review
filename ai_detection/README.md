# AI-Generated Essay Detection

A series of interpretable tests for identifying potentially AI-generated admissions essays. Results are designed to flag essays for qualitative review,
not to make automated determinations.

## Approach

Rather than relying solely on a trained classifier, the primary signal here is **pairwise
cosine similarity against a genre-specific synthetic corpus** — a set of 900 LLM-generated
admissions essays produced under controlled conditions (see `data/synthetic/`).

The motivation: exploratory analysis revealed substantial redundancy in the synthetic data.
Essays generated by the same models cluster tightly in embedding space. Rather than measuring
distance to a centroid, Test 1 computes the maximum cosine similarity between each real essay
and any individual essay in the synthetic corpus, then flags essays that exceed a calibrated
threshold (`NN_THRESHOLD`, default 0.85). This pairwise approach is more sensitive to
near-duplicate matches than aggregate similarity, and makes the flag auditable: a flagged
essay can be shown alongside its closest synthetic neighbor.

The remaining tests (perplexity, stylometrics, discourse) serve as corroborating signals
rather than primary detectors, and are weighted accordingly.

## Tests

| # | Test | Signal | Weight |
|---|---|---|---|
| 1 | Pairwise cosine similarity to synthetic corpus | Does this essay have a near-duplicate match in our synthetic pool? | 0.25 |
| 2 | GPT-2 perplexity + burstiness | LLMs produce uniformly low-perplexity text; humans are "bursty" | 0.30 |
| 3 | Stylometric flatness | High vocabulary richness + low sentence-length variation = LLM signature | 0.30 |
| 4 | Discourse patterns | LLMs produce overly coherent, more argument-dense essays | 0.15 |

Weights are initial estimates based on the literature; revise after running
the discriminability analyses in `essay_assessment/stylometrics.ipynb` (Section 8)
and `essay_assessment/discourse_analysis.ipynb` (Section 6).

## Prerequisites

Before running `ai_detection.ipynb`, the following outputs must exist:

| File | How to produce |
|---|---|
| `essay_assessment/essay_assessment/stylometrics_outputs/stylometric_features.csv` | Run `stylometrics.ipynb` with all three corpora |
| `essay_assessment/essay_assessment/discourse_outputs/full_features.csv` | Run `discourse_analysis.ipynb` with `DATASET = 'real'` |
| `essay_assessment/discourse_outputs_synthetic/full_features.csv` | Run `discourse_analysis.ipynb` with `DATASET = 'synthetic'` |
| `synthetic_data/synthetic_essays.csv` | Already exists (900 essays) |
| `synthetic_data/freeform_essays.csv` | Already exists (600 essays) |

## Outputs

| File | Contents | Privacy |
|---|---|---|
| `outputs/detection_flags.csv` | Per-essay risk scores + flags, includes `student_id` | **Internal only — gitignored** |
| `outputs/detection_summary.csv` | Aggregate statistics only, no `student_id` | Shareable |
| `outputs/perplexity_checkpoint.csv` | GPT-2 inference checkpoint (resume support) | gitignored |
| `outputs/test_summaries/` | Per-test corpus-level statistics | Shareable |

## Privacy Policy

- `detection_flags.csv` is gitignored and must not be shared externally
- No individual student IDs appear in any shareable output
- All aggregate outputs use score distributions, cluster breakdowns, and percentiles only
- Essays are never written to disk during processing

## Notes

- `student_id` prefixes encode corpus: `rea_` = real, `synsyn_` = synthetic, `freff_` = freeform
- GPT-2 perplexity inference takes ~1–2 hours on GPU for the full 15k corpus; run overnight
- The combined risk score targets ROC-AUC > 0.85 (real vs. all synthetic); see Section 5 for validation
